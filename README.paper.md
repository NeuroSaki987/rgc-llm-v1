## 摘要

我们提出 **Resonant Graph Computation（RGC）**，一种面向语言推理的全新计算架构。与以 token 序列为中心、依赖统一隐状态迭代更新的主流大语言模型不同，RGC 将语言理解与推理建模为在 **动态语义-因果-目标图** 上进行的计算过程。该图中的节点表示实体、命题、约束、假设与子目标，边表示因果、时间、指代、蕴含与冲突等关系。RGC 的计算由两个相互耦合的机制驱动：其一是 **共振传播引擎**，通过相位选择性的局部传播在图上实现相关结构的同步与抑制；其二是一个由 **图演算器族** 组成的可学习求解系统，包括合并、分裂、推断、冲突消解、压缩与终止等操作，并由 **元调度器** 在自适应算力预算下按需调用。

在此基础上，RGC 进一步引入 **双时间尺度解码机制**：快速路径用于在低延迟场景下从当前图状态直接提取局部答案草案，深度路径则在高不确定度或高冲突状态下触发多轮图演化与验证，以获得更稳健的推理结果。我们将该架构实现为一个支持可微松弛离散图操作的训练框架，并采用联合目标进行优化，包括文本生成、图一致性、操作器学习、调度优化与内生子问题生成。

我们在多跳问答、长上下文理解、约束规划和时延受限回答等任务上评估 RGC。实验目标不是宣称其在所有场景中全面替代现有序列模型，而是验证一种新的计算范式：**语言智能可以不依赖统一序列混合，而建立在动态图、局部共振传播、目标驱动计算与自适应停机之上。** 结果表明，RGC 在推理深度、可解释中间态和质量-时延折衷方面展现出潜在优势，说明图原生、目标驱动的计算路径可能成为下一代推理系统的重要方向。

## 1. 引言

近年来，大语言模型的发展主要沿着一条相对统一的技术路线推进：将输入文本表示为 token 序列，通过若干层统一的神经算子对其进行混合和变换，并以 next-token prediction 作为核心训练目标。这一路线在语言建模、对话生成、代码补全和广泛的文本任务中取得了巨大成功。然而，从结构角度看，当前绝大多数模型仍然共享一个基本前提：**语言理解与推理本质上是对序列隐状态的持续变换**。

这一前提带来了三类限制。第一，推理结构通常被隐式地压缩在高维向量空间中，模型中间态的语义与逻辑组织难以解释，冲突、依赖与子问题之间的关系不易显式维护。第二，复杂推理往往通过“让同一模型跑更久”来近似实现，而不是通过结构化求解、显式问题拆解与自适应资源分配来实现，这使得“快答”和“深思考”之间难以自然统一。第三，简单问题与复杂问题通常经过近似同构的推理路径，缺乏按难度与价值分配计算资源的机制。

本文提出 **Resonant Graph Computation（RGC）**，试图从计算基元层面对这一范式进行重构。我们不再把语言处理看作对一个 token 序列的全局混合，而是将其建模为：**在一个动态生成的语义-因果-目标图上，进行局部共振传播、图演算化约、目标驱动调度和最优停机的过程。** 输入文本首先被事件化为结构种子，随后逐步构建出可执行的动态图表示。图中的节点不是单纯的 token 位置，而是实体、命题、约束、假设、子目标与推理片段；图中的边则编码因果、蕴含、引用、否定、时间依赖和替代关系。

RGC 的核心计算机制由两部分组成。第一部分是 **共振传播引擎**：每个节点不仅具有语义状态向量，还带有可学习的相位状态，信息传播由邻域内节点间的相位对齐与耦合强度控制。这样，只有在语义相关、逻辑兼容且传播时机匹配时，局部结构才会发生同步增强；否则，信息会被抑制或相互抵消。第二部分是 **图演算器族**：包括 Merge、Split、Infer、Resolve、Compress 与 Halt 等操作器，用于在图上执行语义合并、命题拆分、局部推断、冲突消解、推理压缩与停机决策。所有这些操作并非固定串联，而由一个 **元调度器** 根据当前图态、任务紧迫度、不确定度和冲突强度自适应选择。

在此基础上，RGC 实现了 **快速响应与深度思考的结构性分离**。在低复杂度或低时延预算下，模型通过快速解码器直接从高活性子图中抽取答案草案；在不确定度高、冲突强或任务本身要求论证时，则切换至深度路径，在图上执行多轮传播、推断和验证后再输出结果。这一机制使得模型具备“先给出快答，再根据内部信号决定是否深入求解”的能力，而无需把所有请求一视同仁地送入固定深度网络中。

本文的主要贡献如下：

1. 我们提出 **RGC**，一种以动态图为主计算基底的语言推理架构，用动态语义-因果-目标图取代单一序列隐状态。
2. 我们提出一种 **相位选择性的共振传播机制**，在图邻域中进行局部同步与抑制，而不以全局注意力作为核心主干。
3. 我们设计了一个 **可学习图演算器族与元调度器**，实现图上的合并、分裂、推断、冲突消解、压缩与自适应停机。
4. 我们引入 **双时间尺度解码机制**，统一低延迟响应与深度推理，并将其表述为一种质量-时延折衷下的最优停机问题。
5. 我们提出一套对应的训练与实验框架，用于验证动态图原生计算在长链推理、可解释中间态和自适应计算分配上的潜在优势。

我们并不将 RGC 宣称为对现有大语言模型的全面替代，也不将其包装为一个已被充分验证的工业方案。相反，我们将其视为一种 **新的研究方向与计算范式提案**：当语言智能被重新定义为“在结构化问题空间上的目标驱动求解”，模型架构本身也许可以摆脱统一序列混合的约束，走向更加机器特化、可调度和可解释的路径。

## 2. 相关工作

### 2.1 序列中心的神经语言模型

当前主流大语言模型大多建立在序列中心的建模假设之上。无论是基于全局注意力的 Transformer，还是近年来出现的线性注意力、稀疏注意力、循环记忆变体与状态空间模型，它们都以 token 序列为核心对象，并通过统一或近统一的层间算子对序列隐状态进行迭代更新。这些方法在大规模预训练与通用任务迁移方面取得了显著成果，但在结构化推理、计算自适应性和中间状态可解释性方面仍存在局限。RGC 与这些方法的根本差异并不只在于替换某个算子，而在于将“序列混合”替换为“动态图构造与演化”。

### 2.2 图神经网络与结构化推理

图神经网络、知识图谱推理、神经符号系统和程序归纳模型长期探索了结构化表示在推理中的作用。这些研究表明，当任务具有显式关系、规则或操作结构时，图或程序形式往往能带来更强的组合泛化能力。然而，现有大多数图推理系统通常依赖固定图结构、外部构造的知识图谱，或任务特定的符号模板。RGC 的不同之处在于：**图并非外部附加组件，而是语言理解过程本身的主计算对象**；图结构不是静态给定，而是在推理过程中持续生成、修改、压缩和调度。

### 2.3 自适应计算与条件推理

自适应计算时间（Adaptive Computation Time）、早停网络、条件路由、Mixture-of-Experts 和动态深度模型等方法都试图解决“不同输入应消耗不同计算量”的问题。这些方法通常通过动态层数、条件路由或专家激活来调整计算资源。RGC 与之相似之处在于同样追求按需分配算力，但不同之处在于：RGC 调整的不是单纯的深度或子网络选择，而是 **图上的计算拓扑、操作器调用顺序与子问题展开路径**。因此它的自适应性不仅表现为“算多少”，也表现为“怎么算”。

### 2.4 记忆、检索与外部工具

传统上下文窗口本质上提供的是序列回看能力，而检索增强生成（RAG）、外部记忆模块和工具调用方法则尝试引入更持久和可选择的信息访问机制。RGC 兼容这类研究，但采取不同的抽象：记忆不再是“可回放的一段序列”，而是 **图模式与宏节点组成的层级记忆场**。检索被建模为图模式兼容性匹配，而不是单纯基于向量相似度对文档片段进行排序。

### 2.5 快慢思考与双路径推理

近来一些工作尝试通过草稿生成、再评分、树搜索或 deliberation 机制实现“先快后深”的推理模式。RGC 与这些思路在目标上有相通之处，但其实现方式不同：快速路径与深度路径并不是两个彼此独立的序列模型，而是建立在同一动态图状态之上的两种解码模式。快速路径从当前高活性子图中抽取局部答案，深度路径则通过额外的图演化与验证来提高可靠性。因此，RGC 提供的是一种 **共享中间结构的双时间尺度推理机制**。

## 3. 问题形式化

### 3.1 输入、输出与中间状态

给定输入序列 \(x_{1:n}\)，模型目标是生成输出序列 \(y_{1:m}\)。传统语言模型通常将中间表示写为若干层隐状态 \(H^{(1)}, \ldots, H^{(L)}\)。与之不同，RGC 在任意推理时刻 \(t\) 的核心中间态是一个动态图：

\[
\mathcal{G}_t = (\mathcal{V}_t, \mathcal{E}_t, \mathcal{A}_t),
\]

其中：

- \(\mathcal{V}_t\) 表示节点集合；
- \(\mathcal{E}_t\) 表示边集合；
- \(\mathcal{A}_t\) 表示节点与边上的附加状态场。

在本架构中，节点不对应于单个 token，而可以对应于实体、概念、命题、约束、子目标、假设、计划片段或宏观推理块。边则表示因果关系、时间顺序、指代、逻辑蕴含、冲突、替代、引用或约束依赖。

### 3.2 节点状态与边状态

每个节点 \(v_i \in \mathcal{V}_t\) 具有一个高维语义状态向量 \(h_i \in \mathbb{R}^d\)，并附带若干用于调度和推理的显式状态量。我们定义节点完整状态为：

\[
z_i = (h_i, a_i, c_i, u_i, n_i, r_i, q_i),
\]

其中：

- \(a_i\)：活性（activity），表示该节点当前参与推理的强度；
- \(c_i\)：置信度（confidence）；
- \(u_i\)：任务价值（utility），表示该节点与当前外部任务目标的关联程度；
- \(n_i\)：新颖度（novelty），表示该节点提供的新信息量；
- \(r_i\)：冲突度（conflict intensity）；
- \(q_i\)：可解性（solvability），表示继续展开该节点可能获得有效进展的程度。

每条边 \(e_{ij} \in \mathcal{E}_t\) 则定义为：

\[
e_{ij} = (T_{ij}, w_{ij}, \tau_{ij}),
\]

其中：

- \(T_{ij}\)：边类型分布，可对应因果、指代、时间、蕴含、否定、替代等关系；
- \(w_{ij}\)：边强度；
- \(\tau_{ij}\)：传播相位延迟或时间延迟。

### 3.3 推理作为图动力系统

RGC 将推理过程视为在动态图上的离散时间演化。记元调度器在时刻 \(t\) 的动作选择为 \(a_t\)，则图状态演化可表示为：

\[
\mathcal{G}_{t+1} = F(\mathcal{G}_t, a_t, \xi_t),
\]

其中 \(\xi_t\) 表示探索噪声或随机扰动项。

与固定层数网络不同，RGC 不要求所有样本经历相同长度和相同拓扑的计算。不同输入可以在不同的时间步被终止输出，或者触发不同的操作器与不同规模的图展开。

### 3.4 全局优化目标

为了统一推理正确性、覆盖度、冲突控制与计算成本，我们定义图的总能量函数为：

\[
\mathcal{E}(\mathcal{G}) =
\lambda_1 E_{\text{conflict}}
+ \lambda_2 E_{\text{uncertainty}}
+ \lambda_3 E_{\text{redundancy}}
+ \lambda_4 E_{\text{latency}}
- \lambda_5 E_{\text{coverage}}
- \lambda_6 E_{\text{goal-progress}}.
\]

其中：

- \(E_{\text{conflict}}\)：逻辑冲突能量；
- \(E_{\text{uncertainty}}\)：关键节点不确定性；
- \(E_{\text{redundancy}}\)：冗余表达与重复子结构；
- \(E_{\text{latency}}\)：时间与算力成本；
- \(E_{\text{coverage}}\)：问题覆盖度；
- \(E_{\text{goal-progress}}\)：对子目标或主目标的推进程度。

模型的推理过程可以理解为在预算约束下寻求较低能量状态的过程，即：

\[
\min_{\pi, \Omega} \; \mathbb{E}\big[\mathcal{E}(\mathcal{G}_\tau)\big],
\]

其中 \(\tau\) 为终止时刻，\(\pi\) 为元调度策略，\(\Omega\) 表示可调用的图演算器族。

### 3.5 输出目标

在输出层面，我们关心的不仅是文本似然，还包括答案的正确性、一致性和所用计算资源之间的折衷。因而可将整体目标写为：

\[
\max \; \mathbb{E}
\left[
U_{\text{correctness}}
+ U_{\text{coherence}}
+ U_{\text{coverage}}
- \lambda C_{\text{compute}}
\right].
\]

这一形式化表明，RGC 的目标不是单纯提高 token 级预测精度，而是在 **答案质量、推理完整性与资源效率之间进行自适应平衡**。

## 4. 共振图计算架构

### 4.1 事件化输入编码（Eventized Input Encoding）

RGC 的第一步不是直接对 token 做统一嵌入，而是将输入序列解析为一组离散但可学习的“事件”：

\[
\mathcal{S} = \{s_1, s_2, \ldots, s_m\}.
\]

这里的事件并不是固定语法标签，而是模型内部对输入中关键结构变化的抽象。典型事件包括：

- 新实体引入；
- 指代关系建立；
- 逻辑转折或否定；
- 约束声明；
- 目标陈述；
- 条件句或反事实结构；
- 比较关系；
- 时间更新；
- 问题焦点切换。

事件流经过一个结构生成器 \(\Phi\) 后形成初始图种子：

\[
\mathcal{G}_0 = \Phi(\mathcal{S}).
\]

这一阶段的目标不是得到最终精确图，而是构造一个足以支持后续传播、推断与调度的初始结构。与普通 embedding 不同，\(\Phi\) 的输出是图对象而非单纯向量序列。

### 4.2 动态语义-因果-目标图

初始图会随着推理进程持续增长、收缩、重连与压缩。图中的节点类型包括但不限于：

- **实体节点**：表示对象、概念或角色；
- **命题节点**：表示一句陈述、一个事实或一个可验证断言；
- **约束节点**：表示规则、条件、边界或限制；
- **假设节点**：表示临时成立的推理分支；
- **目标节点**：表示外部任务目标或内部生成的子目标；
- **宏节点**：表示经过压缩后的推理块或思维 chunk。

边用于表示节点间的多样关系。例如：

- 因果边：\(A \rightarrow B\) 表示某事件导致另一事件；
- 蕴含边：\(A \Rightarrow B\) 表示某命题蕴含另一命题；
- 冲突边：\(A \leftrightarrow \neg B\) 表示两节点存在显著矛盾；
- 指代边：连接代词与其潜在指向对象；
- 时间边：表示先后顺序或同步关系；
- 替代边：表示候选解释之间的可替换关系。

由于图并非静态对象，它在推理中承担两个角色：一是 **状态空间**，二是 **求解空间**。模型既在图中存储问题结构，也在图上直接进行推理计算。

### 4.3 共振传播引擎（Resonant Propagation Engine）

传统注意力机制通过 \(QK^\top\) 计算序列位置之间的相似度，再用 softmax 对值向量进行聚合。RGC 不将全局注意力作为主计算原语，而采用图上的相位选择性共振传播。

每个节点不仅具有语义向量 \(h_i\)，还带有一个相位 \(\theta_i\)。我们将节点状态表示为复相位形式：

\[
\psi_i = h_i e^{j\theta_i},
\]

其中 \(j = \sqrt{-1}\)。

在图上进行一次局部传播时，节点 \(i\) 的状态更新为：

\[
\psi_i^{(t+1)} =
\sigma\left(
\sum_{j \in \mathcal{N}(i)}
\kappa_{ij}^{(t)} \psi_j^{(t)} e^{-j\tau_{ij}}
+ b_i^{(t)}
\right),
\]

其中：

- \(\mathcal{N}(i)\) 为节点 \(i\) 的邻域；
- \(\kappa_{ij}^{(t)}\) 是时刻 \(t\) 的可学习耦合系数；
- \(\tau_{ij}\) 是传播延迟；
- \(b_i^{(t)}\) 为偏置或自激项；
- \(\sigma\) 为在复域上实现或分解为实数域实现的非线性映射。

为了直观刻画传播增强与抑制，我们定义节点间的共振强度：

\[
R_{ij} = w_{ij} \cos(\theta_i - \theta_j - \tau_{ij}).
\]

当 \(R_{ij} > 0\) 时，说明两个节点在语义与传播相位上较为对齐，信息会被增强；当 \(R_{ij} < 0\) 时，说明两者之间存在失配、竞争或抑制关系，信息传播会减弱甚至被抵消。

与全局两两注意力相比，这一机制的核心特征在于：

1. 传播是 **局部的**，主要发生在动态图显式连接的邻域中；
2. 传播是 **选择性的**，由相位与耦合共同控制；
3. 传播天然允许 **多假设竞争、冲突抑制和局部同步**。

从复杂度角度看，若图的有效度数受控且调度器只激活部分子图，则单步传播复杂度近似与活跃边数成线性关系，而非与全序列长度的平方成正比。

### 4.4 图演算器族（Graph Calculus Operators）

仅靠信息传播不足以实现复杂推理，因为真正的语言推理往往需要对结构本身进行显式变换。为此，RGC 定义了一组图演算器：

\[
\Omega = \{\text{Merge}, \text{Split}, \text{Infer}, \text{Resolve}, \text{Compress}, \text{Halt}\}.
\]

这些操作器不是手工规则系统，而是可学习的结构变换模块。

#### 4.4.1 Merge：语义合并

对于语义重复、可归并或高度近似的节点，模型可执行合并操作：

\[
\text{Merge}(v_i, v_j) \rightarrow v_{ij}.
\]

合并决策由：

\[
P_{\text{merge}}(i,j) = \sigma\big(f_m(z_i, z_j, e_{ij})\big)
\]

给出，其中 \(f_m\) 为可学习判别器。Merge 的作用是降低冗余、统一指代与压缩重复结构。

#### 4.4.2 Split：结构拆分

若某个节点同时混合了多个命题、多个约束或多个推理责任，模型可对其执行拆分：

\[
\text{Split}(v_i) \rightarrow \{v_i^{(1)}, \ldots, v_i^{(k)}\}.
\]

Split 使模型能够把复杂陈述拆解为多个可分别验证或传播的子命题，从而提高局部求解精度。

#### 4.4.3 Infer：局部推断

Infer 用于识别局部图模式并生成新节点。若某个子图模式与可学习模板 \(\mathcal{P}\) 匹配，则触发新结论生成：

\[
\mathcal{P}(\mathcal{G}) \Rightarrow v_{\text{new}}.
\]

Infer 可以覆盖如下能力：

- 条件链合成；
- 约束传递；
- 变量绑定；
- 反例构造；
- 类比迁移；
- 局部证明片段生成。

#### 4.4.4 Resolve：冲突消解

图中可能同时存在互相矛盾的命题或证据。我们定义冲突能量：

\[
E_{\text{conflict}} = \sum_{(i,j)\in \mathcal{C}} c_i c_j w_{ij}^{\text{contra}},
\]

其中 \(\mathcal{C}\) 为冲突边集合，\(c_i\) 与 \(c_j\) 为节点置信度。Resolve 操作器通过以下方式降低冲突能量：

- 下调低可信节点权重；
- 触发验证型子目标；
- 保留多分支假设而非强制塌缩；
- 请求外部记忆或证据检索。

#### 4.4.5 Compress：宏节点压缩

长链推理若一直保持细粒度结构，会导致图规模膨胀。Compress 操作器将若干节点和边压缩为一个宏节点：

\[
v_{\text{macro}} = \text{Compress}(v_{i_1}, \ldots, v_{i_k}).
\]

宏节点保留摘要语义、局部证明结果与可回溯指针，使模型既能提升后续传播效率，又能在必要时回展开。

#### 4.4.6 Halt：终止

当当前图状态已足以支持输出，或者继续推理的预期收益低于额外计算成本时，模型可调用 Halt 并停止思考，进入最终输出阶段。

### 4.5 元调度器（Meta-Scheduler）

由于图演算器之间不是固定串联，因此需要一个上层控制机制决定何时调用哪种操作。RGC 为此引入元调度器。

首先，模型从图状态中提取一个全局摘要：

\[
g_t = \text{Pool}(\mathcal{G}_t),
\]

其中 \(\text{Pool}\) 可结合节点活性、目标覆盖度、冲突强度和图拓扑统计信息。

元调度策略定义为：

\[
a_t \sim \pi(a \mid g_t),
\]

其中动作 \(a_t\) 包括：

- 调用某个传播步骤；
- 对某一子图进行 Infer；
- 对某一节点进行 Split；
- 对冲突子图调用 Resolve；
- 从记忆场中检索相关宏节点；
- 切换到快速解码路径；
- 切换到深度推理路径；
- 停止思考并输出。

调度器的目标并非单纯最大化最终答案质量，而是优化质量、覆盖与延迟之间的综合收益：

\[
\max_{\pi} \mathbb{E}\left[
U_{\text{correctness}}
+ U_{\text{coverage}}
+ U_{\text{coherence}}
- \lambda C_{\text{latency}}
\right].
\]

这使得 RGC 具备如下特征：

- 简单问题少算；
- 复杂问题多算；
- 有冲突的问题优先验证；
- 有时间压力的问题优先快答；
- 可根据任务类型改变计算拓扑。

### 4.6 层级记忆场（Hierarchical Memory Field）

与序列上下文窗口不同，RGC 采用分层记忆场来存储和访问历史结构：

- **短时记忆（STM）**：当前活跃子图；
- **中时记忆（MTM）**：近期形成的宏节点与局部推理片段；
- **长时记忆（LTM）**：稳定概念图、世界模型局部结构、工具使用模式等。

记忆检索并不被建模为“回看一段旧序列”，而是图模式匹配问题。给定查询模式 \(q\)，检索定义为：

\[
\text{Retrieve}(q) = \arg\max_{m\in \mathcal{M}} \Gamma(q,m),
\]

其中 \(\Gamma\) 是模式兼容性函数。该函数考虑的不仅是语义相似度，还包括节点类型、关系结构、目标相容性和局部拓扑形态。

### 4.7 内生目标场与子问题生成（Intrinsic Goal Field）

为了实现机器特化的“自驱推理”，RGC 不引入拟人化情感或欲望，而是定义一个形式化的目标驱动力量场。对每个节点 \(v_i\)，其综合驱动力定义为：

\[
d_i = \alpha u_i + \beta(1-c_i) + \gamma r_i + \delta n_i + \zeta \Delta_i,
\]

其中：

- \(u_i\)：该节点对主任务的价值；
- \(1-c_i\)：未知程度；
- \(r_i\)：冲突度；
- \(n_i\)：新颖度；
- \(\Delta_i\)：该节点对全局答案质量的边际影响。

当 \(d_i\) 超过阈值 \(\theta_d\) 时，模型会自动生成一个子目标：

\[
\text{if } d_i > \theta_d,\quad g_k = \text{SpawnGoal}(v_i).
\]

子目标可具有如下形式：

\[
g_k = (\text{target node}, \text{objective type}, \text{budget}, \text{deadline}),
\]

其中 objective type 可能包括：

- 解释（explain）；
- 验证（verify）；
- 反例搜索（search counterexample）；
- 缺口补全（fill knowledge gap）；
- 细节展开（expand detail）；
- 计划细化（refine plan）。

这一机制的含义不是“模型拥有类似人的主观欲望”，而是：**模型会在内部状态中发现高价值缺口，并自发形成中间求解目标。**

### 4.8 双时间尺度解码（Dual-Time-Scale Decoding）

为兼顾快速响应与深度思考，RGC 引入两个解码器。

#### 4.8.1 快速路径

快速路径直接从当前图状态中读取高活性子图并生成答案草案：

\[
y_{\text{fast}} = D_{\text{fast}}(\mathcal{G}_t).
\]

快速路径适用于：

- 简单问题；
- 时延预算较小的场景；
- 冲突低、不确定性低的状态；
- 交互式草答。

#### 4.8.2 深度路径

深度路径在当前图上继续执行 \(K\) 轮传播、推断、消解与压缩：

\[
\mathcal{G}_{t}^{(0)} \to \mathcal{G}_{t}^{(1)} \to \cdots \to \mathcal{G}_{t}^{(K)},
\]

之后再生成答案：

\[
y_{\text{deep}} = D_{\text{deep}}(\mathcal{G}_{t}^{(K)}).
\]

#### 4.8.3 融合机制

最终输出概率定义为：

\[
p(y_t) = \rho_t p_{\text{fast}}(y_t) + (1-\rho_t)p_{\text{deep}}(y_t),
\]

其中融合系数 \(\rho_t\) 为：

\[
\rho_t = \sigma\left(
\omega_1 \cdot \text{urgency}
- \omega_2 \cdot \text{uncertainty}
- \omega_3 \cdot \text{conflict}
\right).
\]

也就是说，当任务时间紧迫且当前图态稳定时，模型更偏向快速路径；当不确定度和冲突较高时，模型更倾向深路径。

## 5. 学习与优化

### 5.1 总体训练目标

由于 RGC 同时涉及文本生成、结构构图、图操作选择、调度决策和子目标形成，因此不能仅用标准 next-token prediction 训练，否则模型可能退化为一个复杂但本质上仍然序列化的生成器。我们采用如下联合目标：

\[
\mathcal{L} =
\lambda_1 \mathcal{L}_{\text{text}}
+ \lambda_2 \mathcal{L}_{\text{graph-consistency}}
+ \lambda_3 \mathcal{L}_{\text{operator}}
+ \lambda_4 \mathcal{L}_{\text{scheduler}}
+ \lambda_5 \mathcal{L}_{\text{subgoal}}.
\]

### 5.2 文本生成损失

文本层面的目标仍然是条件语言建模：

\[
\mathcal{L}_{\text{text}} = -\sum_t \log p(y_t \mid \mathcal{G}_t).
\]

这部分确保模型能够将图状态转换为自然语言输出，并与传统生成任务兼容。

### 5.3 图一致性损失

为了约束图中的关系预测和结构语义，我们定义图一致性损失：

\[
\mathcal{L}_{\text{graph-consistency}}
=
\sum_{(i,j)}
\ell_{\text{rel}}(\hat e_{ij}, e_{ij}^{*})
+ \sum_i \ell_{\text{node}}(\hat z_i, z_i^{*}),
\]

其中：

- \(\hat e_{ij}\) 为预测边关系；
- \(e_{ij}^{*}\) 为监督或伪监督目标；
- \(\hat z_i\) 与 \(z_i^{*}\) 分别表示预测与目标节点属性。

在缺少完整人工图标注时，可采用如下弱监督来源：

- 依赖句法/语义解析器的蒸馏标签；
- 信息抽取系统的关系候选；
- 自监督一致性约束；
- 从长链推理结果回溯得到的伪标签。

### 5.4 图演算器训练

图演算器中很多操作本质上是离散选择，例如合并哪两个节点、是否拆分某节点、是否触发某模板推断。为使整个系统可端到端训练，我们对这些离散决策采用可微松弛：

\[
a = \text{softmax}\left(\frac{\log \pi + g}{\tau}\right),
\]

其中 \(g\) 是 Gumbel 噪声，\(\tau\) 为温度参数。对于节点匹配、重排或聚合，也可采用 Sinkhorn 松弛或 straight-through 估计器。

演算器监督损失定义为：

\[
\mathcal{L}_{\text{operator}} = \sum_t \ell(a_t, a_t^{*}),
\]

其中 \(a_t^{*}\) 可以来自人工标注、程序生成任务、启发式轨迹或最终结果反传得到的信用分配信号。

### 5.5 元调度器训练

元调度器负责选择传播、推断、冲突处理和终止等高层动作，因此其优化更适合采用强化学习或 actor-critic 形式。调度器的 reward 可定义为：

\[
R =
\text{correctness}
+ \mu \cdot \text{depth-gain}
+ \nu \cdot \text{coverage}
- \alpha \cdot \text{latency}
- \beta \cdot \text{wasted-compute}.
\]

其中：

- correctness 反映最终答案质量；
- depth-gain 反映额外推理是否真的改进结果；
- coverage 反映问题覆盖程度；
- latency 与 wasted-compute 反映额外计算成本。

于是调度器目标为：

\[
\mathcal{L}_{\text{scheduler}} = - \mathbb{E}_{\pi}[R].
\]

### 5.6 子目标学习

为了鼓励模型在真正存在信息缺口和价值空洞的位置生成有用子目标，而不是无意义地发散，我们定义子目标损失为：

\[
\mathcal{L}_{\text{subgoal}}
=
-\sum_k \log P(g_k^{*}\mid \mathcal{G})
+ \lambda \cdot \text{Penalty}_{\text{useless-goals}}.
\]

这里 \(g_k^{*}\) 可来自以下来源：

- 人工标注的中间求解步骤；
- 规划任务中的分解轨迹；
- 自动生成的验证/反例/补全型中间目标；
- 从成功推理路径中抽取的高价值中间节点。

### 5.7 课程训练策略

由于 RGC 的训练目标耦合度高，直接端到端训练容易不稳定。我们采用分阶段课程学习策略：

1. **事件化与构图预训练**：先训练事件识别与初始图构建；
2. **图演算器预训练**：在结构化合成任务上训练 Merge / Split / Infer / Resolve；
3. **文本解码对齐**：训练从图状态到输出文本的映射；
4. **端到端联合微调**：将图构造、传播、演算与解码联合训练；
5. **调度器强化学习**：在时延预算和难度可变的任务上训练自适应调度；
6. **长程推理强化**：在长上下文、多跳与规划任务上进一步优化深路径能力。

这种课程安排的核心目的是避免系统在早期把所有结构模块退化成噪声通路，并保证各模块先学会局部职责，再进入整体协同。

## 6. 实验设计

### 6.1 研究问题

本文实验围绕以下四个核心研究问题展开：

- **RQ1：动态图表示是否优于纯序列隐状态，尤其在多跳与长链推理任务中？**
- **RQ2：相位共振传播是否优于普通局部消息传递？**
- **RQ3：图演算器与元调度器能否提升计算资源利用效率，使模型对难题多算、对简单题少算？**
- **RQ4：双时间尺度解码能否改善答案质量与响应时延之间的折衷？**

### 6.2 任务设置

为全面评估 RGC，我们使用四类任务。

#### 6.2.1 多跳问答与组合推理

该类任务要求模型跨多个证据源进行组合推断，适合评估子问题生成与局部推断能力。候选基准包括：

- HotpotQA
- 2WikiMultiHopQA
- MuSiQue

这些任务主要考察：

- 跨文档证据整合；
- 中间命题生成；
- 冲突证据处理；
- 组合推理深度。

#### 6.2.2 长上下文理解与文档推理

该类任务用于测试图压缩、宏节点形成和长距离依赖保持能力。候选基准包括：

- NarrativeQA（长文版本）
- Qasper
- GovReport QA
- 长上下文合成依赖任务（synthetic long-range dependency tasks）

主要评估：

- 随上下文长度增长的性能衰减；
- 长链信息保留能力；
- 压缩与回展开的有效性。

#### 6.2.3 规划与约束满足

该类任务强调目标节点、约束传播和中间计划形成。候选基准包括：

- 文本规划 benchmark
- 小规模约束满足任务
- 多条件指令跟随任务
- 基于环境文本反馈的轻量交互式规划任务

评估重点包括：

- 计划成功率；
- 约束违反率；
- 子目标合理性；
- 错误恢复能力。

#### 6.2.4 时延受限回答与 anytime 推理

为了验证双路径机制与调度器，我们构建一个时延受限回答设置：给定不同推理预算（如操作步数、wall-clock budget 或 FLOPs budget），比较模型在各预算下的输出质量。该设置重点考察：

- 快速路径初答质量；
- 深路径增益；
- 调度器的最优停机能力；
- 质量-时延曲线。

### 6.3 基线模型

为确保比较公平，我们设置三类基线。

#### 6.3.1 序列模型基线

- Transformer decoder baseline
- Efficient / sparse / linear attention baseline
- Recurrent or state-space baseline

这些模型代表现有序列中心方法的主流实现。

#### 6.3.2 图增强基线

- GNN over extracted semantic graph
- graph-augmented transformer
- graph memory + sequence decoder baseline

这些基线用于验证：仅仅“加图”是否足够，还是必须采用图原生计算与调度机制。

#### 6.3.3 自适应计算基线

- early-exit transformer
- adaptive computation time variants
- deliberate decoding / tree-style search baseline

这些模型用于比较 RGC 在动态分配算力方面的实际收益。

### 6.4 参数与预算公平性

由于 RGC 属于新范式架构，实验中必须尽可能控制如下变量：

- 参数规模接近；
- 训练 token 数接近；
- 训练总 FLOPs 接近；
- 推理时预算明确可比；
- 统计 wall-clock time 与有效计算量；
- 在相同任务上使用相同数据预处理和同等监督信息。

我们特别报告如下信息：

- parameter count；
- train tokens；
- average active nodes / active edges；
- average operator calls per sample；
- average inference FLOPs；
- median latency；
- worst-case latency。

### 6.5 评估指标

#### 6.5.1 主任务指标

- EM / F1（问答）
- Accuracy（推理分类）
- Success Rate（规划）
- Constraint Satisfaction Rate（约束满足）
- ROUGE / task-specific metrics（长文摘要问答）

#### 6.5.2 推理质量指标

- contradiction rate；
- support consistency；
- intermediate validity；
- explanation faithfulness。

#### 6.5.3 计算效率指标

- latency；
- FLOPs；
- operator calls；
- active subgraph size；
- quality under fixed budget。

#### 6.5.4 可解释性指标

- graph sparsity；
- subgoal usefulness precision；
- macro-node compression ratio；
- human judgment on intermediate trace usefulness。

## 7. 主要实验结果（论文预期结构）

> 注：本节先给出论文中应如何组织结果与分析，具体数值将在实现后填写。为了保持论文严谨性，当前版本不虚构实验结果。

### 7.1 主结果表

论文应首先给出主结果表，对比 RGC 与各类基线在多跳问答、长上下文理解、规划和时延受限设置下的表现。表中建议同时列出：

- 主任务分数；
- 参数规模；
- 平均推理时间；
- 平均推理计算量；
- 在固定预算下的质量指标。

如果实验结果支持论文主张，我们预期 RGC 的优势可能主要体现在：

1. 多跳与冲突密集任务上的结构化推理能力；
2. 长上下文中的性能退化更平缓；
3. 在相同预算下获得更好的质量-时延折衷；
4. 中间推理状态更易解释与分析。

### 7.2 质量-时延曲线

除单点结果外，RGC 论文必须给出质量-时延曲线（quality-latency curve）。该曲线以延迟或推理预算为横轴，以任务分数为纵轴。若双路径与元调度器有效，则应观察到：

- 在极低预算下，快速路径提供有竞争力的答案草案；
- 随着预算增加，深路径带来持续的性能提升；
- 在中等预算区间，RGC 的增长斜率优于固定深度模型；
- 在高预算区间，RGC 的收益趋于稳定，体现合理停机。

### 7.3 难度分桶分析

为了证明模型确实学会了“难题多算、简单题少算”，我们按任务难度对样本进行分桶，例如：

- easy；
- medium；
- hard；
- conflict-heavy；
- long-hop。

然后统计每类样本上的：

- 平均操作步数；
- 平均活跃子图大小；
- 平均深路径调用次数；
- 最终任务得分。

理想情况下，应观察到难题上更多的 Split / Infer / Resolve 调用，以及更低的早停率。

### 7.4 长上下文鲁棒性

在长文与长链依赖任务上，我们应报告：

- 上下文长度增长时的性能曲线；
- 宏节点使用比例；
- 压缩前后活跃图大小变化；
- 关键证据节点是否仍保持可达。

如果 Compress 机制有效，RGC 应能在较长上下文中维持更稳定的推理性能，而非简单依赖更多全局扫描。

### 7.5 规划与约束满足表现

对于规划类任务，除最终成功率外，还应分析：

- 约束违反率；
- 无效子目标比例；
- 计划修正次数；
- 失败恢复能力。

这部分结果若成立，将支持“目标节点 + 约束传播 + 子目标生成”确实为复杂行为生成提供了结构优势。

## 8. 消融实验与分析

由于 RGC 架构由多个新部件组成，消融实验是审稿通过的关键。本节必须详细验证各组件的必要性。

### 8.1 去除相位共振，仅保留普通消息传递

在该消融中，我们将共振传播替换为普通实值图消息传递，不使用相位项与共振强度：

\[
\psi_i^{(t+1)}
=
\sigma\left(
\sum_{j \in \mathcal{N}(i)}
\kappa_{ij}^{(t)} h_j^{(t)} + b_i^{(t)}
\right).
\]

通过比较该版本与完整 RGC，可以测试相位选择性传播是否真正有助于：

- 冲突抑制；
- 多假设竞争；
- 局部相关结构同步；
- 长链推理稳定性。

### 8.2 去除图演算器，仅保留静态图传播

在该版本中，图拓扑在初始构造后不再变化，模型只进行传播和解码。该消融可验证：

- Merge 是否减少冗余；
- Split 是否帮助复杂命题拆解；
- Infer 是否提升多跳推理；
- Compress 是否改善长上下文效率。

### 8.3 去除元调度器，改为固定步骤执行

在该版本中，传播与操作器按照固定顺序执行固定步数。此消融将验证调度器是否真的带来：

- 更好的计算分配；
- 更优的质量-时延折衷；
- 更强的任务自适应性。

### 8.4 去除内生子目标生成

在该版本中，模型不能自动 SpawnGoal，只响应外部任务目标。比较该版本与完整模型，可以评估内生子目标在以下方面的作用：

- 验证需求识别；
- 中间缺口补全；
- 深路径增益；
- 规划任务的中间结构质量。

### 8.5 去除双路径解码

该版本仅保留单一路径输出。该消融可验证双路径机制在 latency-sensitive 设置中的必要性，并分析：

- 快速初答质量；
- 深度修正收益；
- 最优停机策略的表现。

### 8.6 去除压缩器

在长上下文和长链推理任务上去掉 Compress，观察：

- 图规模是否显著膨胀；
- 推理时间是否增加；
- 长上下文准确率是否下降；
- 中间状态是否更不稳定。

### 8.7 初始图构造误差敏感性分析

由于事件化构图是整个系统的入口，我们引入不同程度的噪声，研究 RGC 对图构造误差的敏感性。例如：

- 随机删除部分边；
- 注入错误指代边；
- 误拆分或误合并部分节点。

这样可测试图演算器与调度器是否能在一定程度上纠正初始结构误差。

## 9. 进一步分析

### 9.1 计算分配行为分析

为了证明调度器不是随机地调用更多操作，而是真正学习到了“算力应该花在哪里”，我们统计不同样本上的：

- operator 调用分布；
- 平均深路径长度；
- 冲突高峰时的 Resolve 触发频率；
- 低置信度节点附近的 SpawnGoal 比例。

若结果合理，应表现为：

- 简单样本上更多早停与快速路径输出；
- 冲突样本上更多 Resolve 和验证型子目标；
- 长链样本上更多 Compress 与宏节点形成。

### 9.2 子目标质量分析

我们从验证集采样一批复杂样本，对模型生成的子目标进行人工评估，判断它们是否：

- 指向真正的知识缺口；
- 对最终答案有帮助；
- 不是表面重复或无意义扩展；
- 符合任务导向。

可采用的指标包括：

- subgoal precision；
- utility-weighted subgoal recall；
- average downstream gain per subgoal。

### 9.3 冲突消解能力分析

构造带有矛盾证据的任务子集，例如同时给出支持 \(A\) 与支持 \(\neg A\) 的信息。分析完整模型与各消融版本在以下方面的差异：

- 是否能显式保持分支；
- 是否能延迟最终承诺；
- 是否触发额外验证；
- 最终是否减少自信但错误的回答。

### 9.4 宏节点压缩行为分析

对长文任务中形成的宏节点进行可视化，观察：

- 它们是否对应自然的语义 chunk；
- 是否保留了关键逻辑关系；
- 是否支持按需回展开；
- 压缩是否主要发生在高冗余链条而非关键结论节点上。

### 9.5 中间状态可解释性案例研究

选择若干典型样本，展示从输入文本到初始事件流、再到动态图、局部推断、冲突消解、宏节点形成和最终答案的完整轨迹。该部分不是为了证明形式上的“可解释”，而是为了展示 RGC 的中间态是否能够被研究者和工程师真正读懂与分析。

## 10. 理论讨论

RGC 的目标不是建立一个封闭而完美的理论体系，而是为动态图推理提供可分析的数学支架。本节给出若干有限但可信的理论讨论方向。

### 10.1 局部传播复杂度

若在任意时刻，活跃子图满足有界平均度 \(\bar{d}\)，且调度器限制单步激活边数为 \(|E_{\text{active}}|\)，则单次共振传播的计算量满足：

\[
\mathcal{O}(|E_{\text{active}}|).
\]

这意味着在结构稀疏且调度合理的情况下，RGC 不需要像全局注意力那样在每一步都对全序列做两两比较。

### 10.2 冲突能量下降的局部充分条件

若 Resolve 操作器满足以下局部规则：

1. 对低置信冲突节点下调权重；
2. 对高价值高冲突节点优先触发验证；
3. 对无法立即消解的分支保持并行而非强行塌缩；

则在期望意义下，冲突能量：

\[
E_{\text{conflict}}^{(t+1)} \le E_{\text{conflict}}^{(t)}
\]

在局部步骤中不增。严格证明需要更细的调度假设和噪声条件，但这一结论为 Resolve 设计提供了能量视角。

### 10.3 与耦合振子系统的类比

将节点相位视为耦合振子状态，则相位动力学可近似写为：

\[
\frac{d\theta_i}{dt}
=
\omega_i
+
\sum_j K_{ij}\sin(\theta_j - \theta_i - \tau_{ij}),
\]

其中 \(K_{ij}\) 由边强度和局部兼容性决定。该形式与 Kuramoto 型系统有相似性，但不同之处在于：

- 节点振幅与语义状态耦合；
- 图拓扑可变；
- 操作器可改变耦合结构；
- 调度器可控制局部传播范围。

因此，RGC 可以被视为一种 **可学习、可调度、可变拓扑的图振子系统**。

### 10.4 最优停机视角

双路径解码与元调度可以被统一写为最优停机问题。设当前输出收益为 \(V_{\text{now}}\)，继续推理的期望收益为 \(V_{\text{more}}\)，额外计算成本为 \(C_{\text{extra}}\)。则最优停机规则为：

\[
\text{Stop if } V_{\text{now}} \ge V_{\text{more}} - C_{\text{extra}}.
\]

最优终止时刻定义为：

\[
\tau^* = \arg\max_{\tau}\mathbb{E}[R(\tau)].
\]

这为 RGC 的“什么时候停止思考”提供了明确数学框架，而非启发式阈值堆叠。

## 11. 局限性

尽管 RGC 提供了一个与主流序列模型显著不同的研究方向，但其局限性同样明显。

第一，**训练难度极高**。RGC 同时包含动态图构造、离散图操作、元调度器和双路径输出，多目标耦合极易导致优化不稳定。即使采用课程训练，也不能保证所有模块协同收敛。

第二，**工程复杂度远高于标准堆叠网络**。在现有成熟硬件与软件生态中，固定张量流更容易被高效优化，而动态图、结构修改和条件调度会引入额外的实现负担与并行困难。

第三，**初始构图误差可能被放大**。如果事件化阶段把关键依赖关系抽取错误，后续推理路径可能沿着错误图结构展开。虽然演算器与调度器理论上可纠偏，但其能力需要通过实验严谨验证。

第四，**相位共振动力学的稳定性仍是开放问题**。实际系统中需要设计阻尼、归一化与边权约束，否则传播可能出现不稳定震荡、局部塌缩或噪声放大。

第五，**吞吐优势并非自动成立**。尽管理论上局部传播与自适应调度能减少无效计算，但在工程初期，动态图维护和操作器调用本身可能使系统比成熟的 Transformer 更慢。因此，RGC 更适合作为推理型、结构型任务的新范式探索，而不应在当前阶段宣称自己天然适合所有大规模通用生成场景。

最后，本文中的“内生子目标”与“自驱力”均指向形式化的中间目标生成与资源调度机制，不应被误解为拟人化意识或主观意志。

---

## 12. 结论

本文提出了 **Resonant Graph Computation（RGC）**，一种面向语言推理的动态图原生计算架构。与主流序列中心模型不同，RGC 将语言处理重新表述为：**在动态语义-因果-目标图上的共振传播、图演算化约、目标驱动调度与双时间尺度解码。**

这一架构的核心思想在于：

- 用 **图** 而非单一序列隐状态表示问题空间；
- 用 **相位选择性共振传播** 替代全局统一混合；
- 用 **图演算器族** 显式执行合并、拆分、推断、消解与压缩；
- 用 **元调度器** 实现按难度和价值分配算力；
- 用 **快速路径与深度路径** 统一低延迟响应与深推理。

我们强调，RGC 当前应被理解为一种 **研究提案与计算范式**，而非一个已被工业规模验证的通用替代方案。它的意义不在于简单提供一个新的网络模块，而在于提出一个不同的问题定义：**语言智能是否必须依赖统一的序列混合，还是可以建立在结构化问题空间上的目标驱动求解之上。**

如果后续实现与实验能够验证本文提出的关键假设——尤其是动态图原生计算、自适应调度、局部共振传播与子问题生成在复杂推理任务中的价值——那么 RGC 可能为下一代“更像求解系统、而不只是更大的文本预测器”的语言智能架构提供一个可行起点。

### Algorithm 1: RGC Inference

**Input:** text sequence \(x_{1:n}\), compute budget \(B\)  
**Output:** answer \(y\)

1. Eventize input: \(\mathcal{S} \leftarrow \text{EventEncoder}(x_{1:n})\)
2. Build initial graph: \(\mathcal{G}_0 \leftarrow \Phi(\mathcal{S})\)
3. Initialize time step \(t \leftarrow 0\)
4. While budget remains:
   1. Compute global summary \(g_t \leftarrow \text{Pool}(\mathcal{G}_t)\)
   2. Choose action \(a_t \sim \pi(a \mid g_t)\)
   3. If \(a_t = \text{Propagate}\), run resonant propagation on active subgraph
   4. If \(a_t = \text{Infer}\), apply infer operator on selected subgraph
   5. If \(a_t = \text{Merge/Split/Resolve/Compress}\), apply corresponding operator
   6. Compute node drives \(d_i\); spawn intrinsic subgoals where needed
   7. Evaluate fast answer candidate \(y_{\text{fast}}\)
   8. If halt condition satisfied, break
   9. Update \(t \leftarrow t+1\)
5. If deep mode triggered, run additional graph evolution for \(K\) steps and decode \(y_{\text{deep}}\)
6. Fuse fast/deep outputs and return \(y\)
